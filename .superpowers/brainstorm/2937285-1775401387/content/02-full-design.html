1. 시스템 개요

🗂️

메뉴 시스템

  • Header/Footer 네비게이션 통합 관리
  • 게시판, 경매, 쇼핑 등 모든 링크를 DB에서 관리
  • 트리 구조 (무제한 중첩, UI는 2단계)
  • 그룹(groupName)으로 Footer 컬럼 유연 구성
  • 권한 제어 (all / member / admin)
  • 관리자 드래그앤드롭 트리 UI
🧩

위젯 시스템

  • 코드에서 위젯 컴포넌트를 레지스트리에 등록
  • 관리자가 홈 영역(hero/main/sidebar/bottom)에 배치
  • 위젯별 settings JSON으로 세부 설정
  • 새 기능 추가 시 위젯만 등록하면 홈에 표시 가능
  • on/off, 순서 변경, 크기(col-span) 지정

5. 데이터 흐름

메뉴 흐름
1. GET /api/menus?position=header
2. 서버: Menu 테이블에서 트리 조회
3. Header: 메뉴 렌더링 (visibility 필터링)
4. 관리자: PUT /api/admin/menus 순서/내용 변경
위젯 흐름
1. GET /api/home-widgets
2. 서버: HomeWidget 테이블에서 활성 위젯 조회
3. 홈페이지: widgetRegistry에서 컴포넌트 매칭 → 렌더링
4. ��리자: 위젯 배치/설�� 변경

6. API 엔드포인트

// 메뉴 (공개)

DELETE /api/admin/menus/:id — 메뉴 아이템 삭제
PUT /api/admin/menus/reorder — 드래그앤드롭 순서 변경

// 위젯 (공개)

7. 관리자 UI

메뉴 관리 페이지

/admin/menus

- 탭: Header | Footer
- 드래그앤드롭 트리 (dnd-kit 또는 hello-pangea/dnd)
- 인라인 추가/수정/삭제
- 아이콘 선택, URL 입력, 권한 설정
- 실시간 미리보기 (선택사항)

위젯 관리 페이지

/admin/home-widgets

- 4개 영역(hero/main/sidebar/bottom) 시각적 표시
- 위젯을 영역에 드래그 배치
- 위젯 클릭 시 설정 패널 (settings JSON 기반)
- on/off 토글, col-span/row-span 조정
- 레지스트리에 있지만 미배치된 위젯 목록 ���시
