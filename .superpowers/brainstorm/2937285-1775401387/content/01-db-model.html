DB 모델: Menu 테이블

단일 테이블로 사이트 전체 네비게이션을 관리하는 구조

Prisma Schema

prisma/schema.prisma — Menu model
model Menu { id Int @id @default(autoincrement()) parentId Int? // null = 최상위 메뉴 position String @db.VarChar(20) // "header" | "footer" groupName String? @db.VarChar(50) // Footer 컬럼명, Header 섹션명 label String @db.VarChar(100) // 표시 텍스트 url String @db.VarChar(500) // 링크 URL icon String? @db.VarChar(50) // 이모지 또는 아이콘명 target String @default("_self") @db.VarChar(10) // "_self" | "_blank" visibility String @default("all") @db.VarChar(10) // "all" | "member" | "admin" isActive Boolean @default(true) sortOrder Int @default(0) createdAt DateTime @default(now()) updatedAt DateTime @updatedAt parent Menu? @relation("MenuTree", fields: [parentId], references: [id], onDelete: Cascade) children Menu[] @relation("MenuTree") @@index([position, sortOrder]) @@index([parentId]) @@map("menus") }

핵심 설계 포인트

🌳 트리 구조

parentId 자기참조로 무제한 중첩 가능. onDelete: Cascade로 부모 삭제 시 자식도 함께 삭제

📍 위치 분리

position으로 header/footer 구분, groupName으로 Footer 컬럼이나 Header 섹션을 유연하게 그룹핑

🔐 권한 제어

visibility: "all" (모두), "member" (로그인), "admin" (관리자) — 프론트에서 필터링

🎨 표시 옵션

icon (이모지/아이콘명), target (새창 열기), isActive (표시/숨김) 지원
