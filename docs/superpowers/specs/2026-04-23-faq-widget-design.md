# FAQ Widget Design

**Date:** 2026-04-23
**Status:** Design approved, ready for implementation plan
**Plugin:** `faq-accordion`

## 1. Problem

FAQ accordion plugin은 전용 페이지(`/faq-accordion`)만 제공한다. 랜딩 페이지나 다른 페이지의 레이아웃에서 "자주 묻는 질문" 섹션으로 자연스럽게 노출하려면 별도의 위젯이 필요하다.

## 2. Goals

- 레이아웃 시스템에서 꽂을 수 있는 FAQ 위젯 제공
- 인라인 아코디언으로 방문자가 위젯 안에서 바로 답변 확인
- 매 10분마다 새로 랜덤 선별해 정체 방지
- 하단 "전체 FAQ 보기" 링크로 전용 페이지 유도
- Admin이 위젯 설정(`limit`)으로 표시 개수 조절

## 3. Non-Goals

- 위젯 내 카테고리 필터·검색 (전용 페이지에서만)
- 위젯 내 도움됨/도움 안 됨 투표 (전용 페이지에서만)
- "특정 카테고리만" 등 세밀한 admin 설정
- 조회 수 기반 인기순 — 정체가 생겨 랜덤 로테이션을 선택

## 4. Placement / Default Layout

- `defaultZone: 'main'`
- `defaultColSpan: 6`
- `defaultRowSpan: 2`

Admin은 레이아웃 편집기에서 zone/span 자유 조정 가능.

## 6. API — GET `/api/faq-accordion`

**Query params:**
- `type=random` (필수, 다른 값이면 400)
- `limit=<number>` (선택) — 1~10 범위로 clamp, 기본 5

**Response (200):**
```json
{ "faqs": [{ "id": 3, "question": "…", "answer": "<p>…</p>", "views": 12 }] }
```

- `published=true`인 FAQ만 대상
- MySQL `ORDER BY RAND() LIMIT N` 사용
- Prisma는 `RAND()`를 직접 표현하지 못하므로 `$queryRaw<{id: bigint}[]>`으로 id만 랜덤 뽑고 뒤이어 `findMany({ where: { id: { in: ids } } })`로 본문 취득

**Caching:**
- `unstable_cache(fn, ['faq-random', String(limit)], { revalidate: 600 })`
- 10분 TTL → 같은 `limit`에 대해 10분간 같은 선택 반환
- 10분 경과 후 다음 요청에 재생성

**Errors:**
- `type`이 `random` 외: `400 { error: 'Invalid type' }`
- `limit` 비정상: clamp 적용 (에러 없이 1~10 범위로 보정)
- 서버 예외: `500 { error: 'Internal error' }`

## 7. Widget Component — `FaqWidget.tsx`

**Props:** `{ settings?: { limit?: number } }` (widget system 규약)

**Fetching:**
- 마운트 시 `fetch('/api/faq-accordion?type=random&limit=' + (settings?.limit ?? 5))`
- 로딩 중: 스켈레톤 3줄 (회색 블록)
- 응답 비어있음: `<div>{t('empty')}</div>`
- 에러: 동일하게 `{t('empty')}` (위젯은 조용히 숨는 게 자연스러움)

**Rendering:**
- `<Card>` 컨테이너
- 상단: 타이틀 `{t('title')}` ("자주 묻는 질문")
- 본문: 공개 페이지의 `Accordion` / `AccordionItem` 프리미티브 재사용
  - 각 아이템: 질문 + 조회수 (Eye 아이콘)
  - 펼치면 `sanitizeHtml(answer)`로 정제된 HTML을 `prose prose-sm dark:prose-invert`로 렌더
- 펼침 시 `PATCH /api/faq-accordion` (기존, `type: 'view'`) 호출 — 조회수 증가, 응답으로 새 views 반영
- 투표 UI 없음
- 하단: `<Link href="/faq-accordion">{t('widget.viewAll')} →</Link>`

**Styling:**
- 카드 패딩, 내부 요소 세로 간격은 공개 페이지 스타일과 맞춤 (`space-y-3`~`space-y-4`)

## 8. Widget Metadata — `FaqWidget.meta.ts`

```ts
export default {
  title: 'FAQ',
  defaultZone: 'main',
  defaultColSpan: 6,
  defaultRowSpan: 2,
  settingsSchema: { limit: 5 },
}
```

스캐너가 자동으로 `_generated-registry.ts` / `_generated-metadata.ts`에 포함. `plugin.ts` 변경 없음.

## 9. i18n

## 10. Testing / Verification

수동 검증 체크리스트:

- [ ] 위젯이 admin 레이아웃 편집기의 위젯 선택 목록에 "FAQ"로 나타남
- [ ] 위젯을 레이아웃에 추가한 페이지에서 아코디언 5개(기본) 노출
- [ ] 아이템 펼치면 답변이 sanitize된 HTML로 렌더링
- [ ] 펼치는 순간 조회수 +1 (새로고침해도 유지)
- [ ] 투표 버튼이 없음 (공개 페이지에만 있음)
- [ ] 하단 "전체 FAQ 보기" 클릭 → `/faq-accordion`
- [ ] 위젯 설정에서 `limit`을 3으로 바꾸면 3개 노출
- [ ] 10분 이내 여러 번 새로고침해도 같은 5개 (캐시)
- [ ] 10분 이후 재방문 시 다른 조합 (새 랜덤)
- [ ] 공개된 FAQ가 0개일 때 위젯이 빈 메시지 표시
- [ ] 공개된 FAQ가 limit 미만일 때 가능한 만큼만 노출 (에러 없음)
