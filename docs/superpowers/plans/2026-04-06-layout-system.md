# Layout System Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Allow admin to switch between default and custom layouts (Header/HomePage/Footer) with partial override support.

**Architecture:** A scan script generates a manifest of available layout folders at build time. A layout loader uses `next/dynamic` with the manifest to load components with fallback to default. Admin selects the active layout folder via the existing Settings system.

**Tech Stack:** Next.js dynamic imports, Node.js fs (scan script), existing Prisma Setting model, ShadCN UI components.

### Task 1: Create scan-layouts script

**Files:**
- Create: `scripts/scan-layouts.js`

- [ ] **Step 1: Create the scan script**

```javascript
// scripts/scan-layouts.js
const fs = require('fs')
const path = require('path')

const LAYOUTS_DIR = path.join(__dirname, '..', 'src', 'layouts')
const OUTPUT_FILE = path.join(LAYOUTS_DIR, '_generated.ts')
const COMPONENT_NAMES = ['Header', 'HomePage', 'Footer']

function scanLayouts() {
  // Ensure layouts dir exists
  if (!fs.existsSync(LAYOUTS_DIR)) {
    fs.mkdirSync(LAYOUTS_DIR, { recursive: true })
  }

  // Read subdirectories, skip _ prefixed
  const entries = fs.readdirSync(LAYOUTS_DIR, { withFileTypes: true })
  const folders = entries
    .filter(e => e.isDirectory() && !e.name.startsWith('_'))
    .map(e => e.name)

  // Validate default folder exists
  if (!folders.includes('default')) {
    console.error('ERROR: src/layouts/default/ folder is required but missing!')
    process.exit(1)
  }

  // Check default has all 3 files
  for (const comp of COMPONENT_NAMES) {
    const filePath = path.join(LAYOUTS_DIR, 'default', `${comp}.tsx`)
    if (!fs.existsSync(filePath)) {
      console.error(`ERROR: src/layouts/default/${comp}.tsx is required but missing!`)
      process.exit(1)
    }
  }

  // Build manifest
  const manifest = {}
  for (const folder of folders) {
    const files = {}
    for (const comp of COMPONENT_NAMES) {
      files[comp] = fs.existsSync(path.join(LAYOUTS_DIR, folder, `${comp}.tsx`))
    }
    manifest[folder] = { name: folder, files }
  }

  // Build dynamic import map entries
  const importEntries = []
  for (const folder of folders) {
    for (const comp of COMPONENT_NAMES) {
      if (manifest[folder].files[comp]) {
        importEntries.push(`  '${folder}/${comp}': dynamic(() => import('@/layouts/${folder}/${comp}'), { ssr: false })`)
      }
    }
  }

  // Generate TypeScript file
  const output = `// AUTO-GENERATED by scripts/scan-layouts.js — do not edit manually
import dynamic from 'next/dynamic'
import type { ComponentType } from 'react'

export const layoutManifest = ${JSON.stringify(manifest, null, 2)} as const

export type LayoutFolder = keyof typeof layoutManifest
export type LayoutComponent = 'Header' | 'HomePage' | 'Footer'

// eslint-disable-next-line @typescript-eslint/no-explicit-any
export const componentMap: Record<string, ComponentType<any>> = {
${importEntries.join(',\n')}
}
`

  fs.writeFileSync(OUTPUT_FILE, output, 'utf-8')
  console.log(`[scan-layouts] Found ${folders.length} layout(s): ${folders.join(', ')}`)
}

scanLayouts()
```

### Task 3: Create layout loader

**Files:**
- Create: `src/lib/layout-loader.ts`

- [ ] **Step 1: Create the layout loader**

```typescript
// src/lib/layout-loader.ts
import type { ComponentType } from 'react'
import { layoutManifest, componentMap } from '@/layouts/_generated'
import type { LayoutFolder, LayoutComponent } from '@/layouts/_generated'

/**
 * Get a layout component for the given folder and component name.
 * Falls back to 'default' if the component doesn't exist in the specified folder.
 */
// eslint-disable-next-line @typescript-eslint/no-explicit-any
export function getLayoutComponent(folder: string, component: LayoutComponent): ComponentType<any> {
  const validFolder = (folder in layoutManifest) ? folder as LayoutFolder : 'default'
  const manifest = layoutManifest[validFolder]

  // Check if the component exists in the specified folder
  const key = `${validFolder}/${component}`
  if (manifest.files[component] && componentMap[key]) {
    return componentMap[key]
  }

  // Fallback to default
  const defaultKey = `default/${component}`
  if (componentMap[defaultKey]) {
    return componentMap[defaultKey]
  }

  // Should never happen if default folder is valid
  throw new Error(`Layout component not found: ${component}`)
}

/**
 * Get all available layout folder names.
 */
export function getAvailableLayouts() {
  return Object.entries(layoutManifest).map(([key, value]) => ({
    folder: key,
    name: value.name,
    files: value.files,
  }))
}
```

### Task 5: Create layouts API endpoint

**Files:**
- Create: `src/app/api/admin/layouts/route.ts`

- [ ] **Step 1: Create the API route**

```typescript
// src/app/api/admin/layouts/route.ts
import { NextResponse } from 'next/server'
import { getAdminUser } from '@/lib/auth'
import { getAvailableLayouts } from '@/lib/layout-loader'

// GET /api/admin/layouts — list available layout folders
export async function GET() {
  try {
    const admin = await getAdminUser()
    if (!admin) {
      return NextResponse.json({ error: '권한이 없습니다.' }, { status: 401 })
    }

    const layouts = getAvailableLayouts()

    return NextResponse.json({ layouts })
  } catch (error) {
    console.error('레이아웃 목록 조회 에러:', error)
    return NextResponse.json({ error: '서버 오류' }, { status: 500 })
  }
}
```
