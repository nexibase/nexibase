# i18n (next-intl) 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 넥시베이스에 next-intl 기반 국제화(i18n)를 적용하여 영어(기본)/한국어 2개 언어를 `/en/...`, `/ko/...` 경로로 지원한다.

**Architecture:**
- next-intl `always` prefix 모드: 기본 언어(en)도 URL에 prefix 포함 → 기본 언어 변경 시 SEO 영향 최소화
- 모든 라우트를 `src/app/[locale]/` 아래로 이동 (API는 예외, locale 무관)
- scan-plugins.js가 플러그인 래퍼를 `src/app/[locale]/{slug}/` 에 생성하도록 수정
- 플러그인별 `locales/ko.json`, `locales/en.json`을 scan-plugins가 머지하여 `src/messages/{locale}.json` 생성
- 코어 미들웨어(proxy.ts)와 next-intl 미들웨어를 chain 방식으로 결합

**Tech Stack:** next-intl, Next.js 16 App Router, TypeScript

**범위 제한 (Phase 1 - 이 계획의 목표):**
- 인프라 구축: 라우팅, 미들웨어, scan-plugins, 플러그인 locale 시스템
- 코어 핵심 UI(Header, Footer, 언어 전환 버튼) 번역
- 단일 샘플 플러그인(boards) 번역으로 동작 검증
- 관리자 페이지, 다른 플러그인(shop, contents, policies)의 전면 번역은 **별도 계획(Phase 2)** 으로 진행

**Phase 1의 완료 기준:**
- `/` 접속 시 `/en`으로 리다이렉트
- `/en/boards`, `/ko/boards` 정상 작동
- Header/Footer에 언어 전환 버튼, 클릭 시 `/en/...` ↔ `/ko/...` 전환
- 게시판 목록 페이지의 주요 UI 텍스트가 언어별로 표시
- 다른 플러그인(shop/contents/policies)과 관리자는 한국어 유지(번역 안 된 상태에서도 동작)

현재 파일 구조를 유지하되, 기본 export를 named export로 변경:

`src/proxy.ts`의 `export default middleware` 부분을 `export async function pluginMiddleware`로 변경하고, `config`는 그대로 두되 `export const config`는 제거(루트 middleware.ts에서 재정의).

- [ ] **Step 2: 루트 `middleware.ts` 생성**

```typescript
import { NextRequest, NextResponse } from 'next/server'
import createIntlMiddleware from 'next-intl/middleware'
import { routing } from '@/i18n/routing'
import { pluginMiddleware } from '@/proxy'

const intlMiddleware = createIntlMiddleware(routing)

export async function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl

  // API 경로는 locale 처리하지 않고 플러그인 미들웨어만 실행
  if (pathname.startsWith('/api/')) {
    return pluginMiddleware(request)
  }

  // 먼저 locale 처리
  const intlResponse = intlMiddleware(request)

  // next-intl이 리다이렉트/리라이트를 수행한 경우 그대로 반환
  if (intlResponse.headers.get('x-middleware-rewrite') || intlResponse.status === 307 || intlResponse.status === 308) {
    return intlResponse
  }

  // 그 외에는 플러그인 미들웨어 실행
  return pluginMiddleware(request)
}

export const config = {
  matcher: [
    '/((?!_next/static|_next/image|favicon.ico|uploads|themes|.*\\.(?:png|jpg|jpeg|gif|webp|svg|ico|css|js|txt|xml)$).*)',
  ],
}
```

- [ ] **Step 3: `src/app/[locale]/layout.tsx` 생성**

```typescript
import type { Metadata } from 'next'
import { NextIntlClientProvider, hasLocale } from 'next-intl'
import { getMessages, setRequestLocale } from 'next-intl/server'
import { notFound } from 'next/navigation'
import { routing } from '@/i18n/routing'
import { SiteShell } from '@/components/layout/SiteShell'
import { SiteProvider } from '@/lib/SiteContext'

export async function generateStaticParams() {
  return routing.locales.map((locale) => ({ locale }))
}

export const metadata: Metadata = {
  title: 'NexiBase',
  description: 'NexiBase - Community Platform',
}

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode
  params: Promise<{ locale: string }>
}) {
  const { locale } = await params
  if (!hasLocale(routing.locales, locale)) {
    notFound()
  }
  setRequestLocale(locale)
  const messages = await getMessages()

  return (
    <NextIntlClientProvider messages={messages}>
      <SiteProvider>
        <SiteShell>{children}</SiteShell>
      </SiteProvider>
    </NextIntlClientProvider>
  )
}
```

`scripts/scan-plugins.js`의 `mergeSchemas` 함수 뒤에 `mergeLocales` 함수 추가:

```javascript
function mergeLocales(plugins) {
  const CORE_LOCALES_DIR = path.join(__dirname, '..', 'src', 'locales')
  const OUT_DIR = path.join(__dirname, '..', 'src', 'messages')
  const LOCALES = ['ko', 'en']

  if (!fs.existsSync(OUT_DIR)) {
    fs.mkdirSync(OUT_DIR, { recursive: true })
  }

  for (const locale of LOCALES) {
    const merged = {}

    // 1. 코어 메시지 로드
    const corePath = path.join(CORE_LOCALES_DIR, `${locale}.json`)
    if (fs.existsSync(corePath)) {
      Object.assign(merged, JSON.parse(fs.readFileSync(corePath, 'utf-8')))
    }

    // 2. 각 플러그인 메시지 머지
    for (const p of plugins) {
      const pluginLocalePath = path.join(PLUGINS_DIR, p.folder, 'locales', `${locale}.json`)
      if (fs.existsSync(pluginLocalePath)) {
        const pluginMessages = JSON.parse(fs.readFileSync(pluginLocalePath, 'utf-8'))
        Object.assign(merged, pluginMessages)
      }
    }

    // 3. 출력
    const outPath = path.join(OUT_DIR, `${locale}.json`)
    fs.writeFileSync(outPath, JSON.stringify(merged, null, 2), 'utf-8')
  }

  console.log(`[scan-plugins] Merged locales for ${LOCALES.length} language(s)`)
}
```

## Task 6: 언어 전환 컴포넌트 (LocaleSwitcher)

**Files:**
- Create: `src/components/layout/LocaleSwitcher.tsx`

- [ ] **Step 1: `LocaleSwitcher.tsx` 생성**

```typescript
'use client'

import { useTranslations, useLocale } from 'next-intl'
import { useRouter, usePathname } from '@/i18n/navigation'
import { routing } from '@/i18n/routing'
import { Globe } from 'lucide-react'
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuTrigger,
} from '@/components/ui/dropdown-menu'
import { Button } from '@/components/ui/button'

export function LocaleSwitcher() {
  const t = useTranslations('locale')
  const locale = useLocale()
  const router = useRouter()
  const pathname = usePathname()

  const switchTo = (nextLocale: string) => {
    router.replace(pathname, { locale: nextLocale as 'ko' | 'en' })
  }

  return (
    <DropdownMenu>
      <DropdownMenuTrigger asChild>
        <Button variant="ghost" size="icon" aria-label={t('switchTo')}>
          <Globe className="h-4 w-4" />
        </Button>
      </DropdownMenuTrigger>
      <DropdownMenuContent align="end">
        {routing.locales.map((loc) => (
          <DropdownMenuItem
            key={loc}
            onClick={() => switchTo(loc)}
            className={loc === locale ? 'font-semibold' : ''}
          >
            {t(loc)}
          </DropdownMenuItem>
        ))}
      </DropdownMenuContent>
    </DropdownMenu>
  )
}
```
